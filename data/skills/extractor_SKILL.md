# 角色场景提取器 — 专业技能指南

## 核心工作流程

1. **读取剧本**：调用 `read_script_for_extraction` 获取剧本内容
2. **检查已有数据**：调用 `read_existing_characters` 和 `read_existing_scenes` 获取已有记录
3. **提取角色**：分析剧本，提取所有角色的详细信息
4. **提取场景**：分析剧本，提取所有场景的描述
5. **保存结果**：调用 `save_characters` 和 `save_scenes` 保存（自动去重）

## 角色提取技巧

### 角色识别
- 不仅有名字的角色需要提取，有对白但未命名的人物也应记录（如"服务员"、"路人甲"）
- 从动作描写中推断角色信息（如"他摸了摸花白的头发"→ 年龄偏大）
- 从对白语气推断性格和声音特征

### 角色分级
- **主角**：出场最多、推动核心剧情的角色
- **反派**：与主角有直接冲突的角色
- **配角**：有固定名字和多次出场的角色
- **龙套**：只出现一两次、对剧情影响小的角色

### 去重策略
- 同名角色一定合并
- 别名/绰号指向同一人时合并（如"老张"和"张建国"）
- 合并时保留更完整的信息

## 场景提取技巧

### 场景识别
- 从场景标题（如"内景-咖啡馆-日"）直接提取
- 从叙述中的地点和时间描述推断
- 同一地点不同时间是不同场景

### Prompt生成要点
- 使用英文，关键词格式
- 包含：风格 + 场景类型 + 光线 + 色调 + 构图
- 避免抽象描述，使用具体视觉词汇
- 示例："Cinematic wide shot, cozy vintage cafe interior, warm golden afternoon light, amber color palette, wooden furniture and bookshelves, 8k ultra detailed"

## 常见问题处理

- **角色信息不全**：从上下文推断，不确定的留空
- **场景模糊**：根据描写推断最可能的地点和时间
- **大量角色**：按出场重要性排序，确保主要角色信息完整
