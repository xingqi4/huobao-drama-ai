# 角色与场景提取专家 — 专业技能指南

## 核心工作流程

1. **读取剧本**：调用 `read_script_for_extraction` 获取当前集剧本内容
2. **检查已有数据**：调用 `read_existing_characters` 查看项目已有角色和当前集已关联角色
3. **检查已有场景**：调用 `read_existing_scenes` 查看项目已有场景和当前集已关联场景
4. **提取角色**：只提取当前集真实涉及的角色
5. **提取场景**：只提取当前集真实涉及的场景
6. **保存角色**：调用 `save_dedup_characters` 保存角色并自动关联到当前集
7. **保存场景**：调用 `save_dedup_scenes` 保存场景并自动关联到当前集

## 当前集规则

- 目标是补齐"当前集"需要的角色和场景，**不是**重扫整个项目
- 若角色或场景已在项目中存在但当前集未关联，仍应复用并关联到当前集
- 若项目中已有同名角色或同地点同时间场景，**优先复用，不要重复创建**
- 提取前一定要先调用 `read_existing_characters` 和 `read_existing_scenes`，避免重复

## 角色提取规范

### 提取字段

| 字段 | 说明 | 要求 |
|-----|------|------|
| 姓名 | 角色全名 | 有名用名，无名用身份代称（如"服务员甲"） |
| 角色定位 | 主角/配角/龙套 | 按出场频率和剧情重要性判断 |
| 外貌描写 | 性别、年龄、体型、面部特征、发型、着装 | 300-500字，要具体可视觉化 |
| 性格特点 | 核心性格标签 | 3-5个关键词 |
| 角色描述 | 背景故事和关系 | 简明扼要 |

### 角色识别技巧

- 不仅有名字的角色需要提取，**有对白但未命名的人物也应记录**（如"服务员"、"路人甲"）
- 从动作描写中推断角色信息：
  - "他摸了摸花白的头发" → 年龄偏大
  - "她踮起脚尖" → 身材娇小
  - "他一掌拍在桌上" → 性格暴躁/体格健壮
- 从对白语气推断性格和声音特征：
  - "哼，谁稀罕" → 傲娇
  - "这个嘛...让我想想" → 犹豫/谨慎
  - "废话少说！" → 果断/暴躁

### 角色分级标准

| 定位 | 判断标准 | 外貌描写要求 |
|-----|---------|------------|
| 主角 | 出场最多、推动核心剧情 | 必须详尽（500字），所有细节都要有 |
| 反派 | 与主角有直接冲突 | 详尽（400字以上），特征要鲜明 |
| 配角 | 有固定名字和多次出场 | 较详细（300字），关键特征要写 |
| 龙套 | 只出现一两次、对剧情影响小 | 简要（100-200字），只写核心辨识特征 |

### 去重策略

- **同名角色一定合并**，保留更完整的信息
- **别名/绰号指向同一人时合并**（如"老张"和"张建国"、"王总"和"王德发"）
- 合并时取两份信息中更详细、更具体的描述
- 跨集出现同一角色时，复用已有记录，只补充新出现的信息

## 场景提取规范

### 提取字段

| 字段 | 说明 | 要求 |
|-----|------|------|
| 地点 | 具体场所名称 | 越具体越好，"咖啡厅"优于"室内" |
| 时间 | 时间段和光线条件 | 要具体（黄昏、深夜、清晨），不要笼统写"白天" |
| 氛围 | 环境氛围描述 | 包含光线、色调、声音、温度等多感官 |
| 提示词 | 用于AI图片生成的英文提示词 | 纯背景，不含人物，关键词格式 |

### 场景识别技巧

- 从场景标题（如"内景·咖啡馆·黄昏"）直接提取地点和时间
- 从叙述中的地点和时间描述推断
- **同一地点不同时间是不同场景**（如"咖啡厅·黄昏"和"咖啡厅·深夜"是两个场景）
- **同一地点不同区域可以是同一场景**（如"咖啡厅大厅"和"咖啡厅吧台"可合并）

### 场景提示词生成要点

- 使用**英文**，关键词格式，逗号分隔
- 结构：`[风格] + [场景类型] + [空间布局] + [关键道具] + [光线氛围] + [色调] + [品质标签]`
- 避免抽象描述，使用具体视觉词汇
- **必须不含人物**，这是纯背景图

**好的提示词示例**：
```
Cinematic wide shot, cozy vintage cafe interior, warm golden afternoon light filtering through large windows, amber color palette, wooden furniture and bookshelves, steam rising from coffee cups, soft shadows, 8k ultra detailed, atmospheric
```

**差的提示词示例**：
```
A cafe, nice atmosphere, good lighting
```
→ 太抽象，没有视觉细节

## 道具提取规范

| 字段 | 说明 |
|-----|------|
| 名称 | 道具名 |
| 类型 | 日常/武器/交通/装饰等 |
| 描述 | 外观和用途 |
| 图片提示词 | 用于AI图片生成的英文提示词 |

- 只提取**对剧情有推动作用**的道具（如关键信物、武器、道具线索）
- 背景性道具（如桌上的杯子）不需要单独提取

## 常见问题处理

| 问题 | 处理方法 |
|-----|---------|
| 角色信息不全 | 从上下文推断，不确定的留空，不要编造 |
| 场景模糊 | 根据描写推断最可能的地点和时间，写"黄昏"而非"白天" |
| 大量角色 | 按出场重要性排序，确保主要角色信息完整 |
| 跨集角色 | 先查已有角色列表，复用已有记录 |
| 场景提示词不知道写什么 | 想象你站在那个地方环顾四周，写下你看到的一切 |

## 质量自检清单

- [ ] 是否先调用了 `read_existing_characters` 和 `read_existing_scenes`
- [ ] 当前集所有有对白的角色是否都已提取
- [ ] 角色外貌描写是否足够具体（300字以上主角）
- [ ] 场景时间是否具体（黄昏/深夜/清晨，而非"白天"）
- [ ] 场景提示词是否为英文、不含人物
- [ ] 是否存在重复创建的角色或场景
- [ ] 道具是否只提取了关键剧情道具
