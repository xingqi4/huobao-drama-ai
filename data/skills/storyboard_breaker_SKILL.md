# 分镜拆解专家 — 专业技能指南

## 拆解原则

每个镜头聚焦**单一动作**，描述要详尽具体。每个镜头时长 10-15 秒。
核心口诀：**一个镜头讲一件事，一段视频演一个动作**。

## 核心工作流程

1. **读取上下文**：调用 `read_storyboard_context` 获取剧本、角色、场景、已有分镜摘要
2. **整体规划**：先通读剧本，规划镜头总数和节奏，确保总时长和叙事连续性合理
3. **分批拆解**：⚠️ **必须分批保存**，每次只生成3-5个镜头，然后立即调用 `save_storyboards` 保存
   - **第一批**：`save_storyboards(storyboards=[镜头1-5], append=false)` — append=false清除旧数据
   - **后续批次**：`save_storyboards(storyboards=[镜头6-10], append=true)` — append=true追加不覆盖
   - 重复直到所有镜头都保存完毕
4. **按需调整**：如需修改，调用 `update_storyboard` 修改具体镜头

### ⚠️ 分批保存是强制的！
一次性生成所有分镜会导致输出超时或被截断。分批生成确保每批数据完整保存，避免丢失进度。

## 镜头要素（17 个必填/选填字段）

| # | 字段 | 说明 | 示例 |
|---|------|------|------|
| 1 | title | 3-5字概括核心内容 | "噩梦惊醒" |
| 2 | time | 具体时分 + 光线描述 | "凌晨3点，月光透过窗帘" |
| 3 | location | 场景完整描述 + 空间布局 + 环境细节 | "卧室，床靠右墙，左侧是落地窗" |
| 4 | shot_type | 景别 | 远景/全景/中景/近景/特写 |
| 5 | angle | 角度 | 平视/仰视/俯视/侧面/背面 |
| 6 | movement | 运镜 | 固定/推镜/拉镜/摇镜/跟镜/移镜 |
| 7 | action | 谁 + 具体怎么做 + 肢体细节 + 表情 | "小明猛地坐起，额头冒汗，大口喘气" |
| 8 | dialogue | 该镜头的完整对话 | "小明：我做了个噩梦" |
| 9 | result | 动作的即时后果 + 视觉细节 | "被子滑落到腰间，枕头被汗水浸湿" |
| 10 | atmosphere | 光线 + 色调 + 声音 + 整体氛围 | "冷蓝色月光，寂静中只有急促呼吸声" |
| 11 | duration | 时长 | 10-15秒 |
| 12 | image_prompt | 静态画面提示词，用于首帧/尾帧/镜头图片生成 | 英文，突出构图和视觉 |
| 13 | video_prompt | 视频提示词，按3秒分段（**必填**） | 见下方格式 |
| 14 | bgm_prompt | 配乐提示词 | 简洁短语，如"紧张的心跳声渐强" |
| 15 | sound_effect | 音效提示词 | 简洁短语，如"急促呼吸声，被子窸窣" |
| 16 | scene_id | 场景关联 | 从已有场景中匹配 |
| 17 | character_ids | 角色关联 | 绑定涉及的0到多个角色 |

## 分镜设计原则

### 镜头选择的叙事逻辑

| 叙事阶段 | 推荐景别 | 目的 |
|---------|---------|------|
| 开场/建立 | 远景/大远景 | 让观众知道"在哪里" |
| 对白交流 | 中景/过肩镜头 | 展示人物关系和互动 |
| 关键对白 | 近景/特写 | 强调情绪和表情 |
| 动作场面 | 中景跟随 | 保持动作连贯性 |
| 情感高潮 | 特写 | 捕捉微妙表情变化 |
| 场景过渡 | 远景/空镜头 | 给观众"呼吸"空间 |

### 摄影角度的情感含义

| 角度 | 情感含义 | 典型用法 |
|-----|---------|---------|
| 仰视 | 权力、威严、压迫感 | 反派登场、主角觉醒 |
| 俯视 | 弱小、无助、客观审视 | 角色受挫、困境 |
| 荷兰角（倾斜） | 不安、失衡、疯狂 | 精神崩溃、真相揭露 |
| 平视 | 自然、平等、日常 | 日常对话、生活场景 |
| 背面 | 孤独、离去、神秘 | 角色远去、背影杀 |

### 运镜方式的选择

| 运镜 | 效果 | 适用场景 |
|-----|------|---------|
| 推镜 | 引导注意力，强调 | 发现关键线索、情绪爆发 |
| 拉镜 | 揭示全貌，疏离 | 真相大白、孤独感 |
| 摇镜 | 跟随动作，展示空间 | 角色走动、环顾四周 |
| 跟镜 | 临场感，紧迫感 | 追逐、逃跑 |
| 移镜 | 流畅叙事，空间转换 | 跟随角色移动 |
| 固定 | 稳定、客观、静默 | 对峙、等待、沉思 |

## 视频提示词格式（video_prompt）

每个镜头**必须**包含 `video_prompt` 字段，按 3 秒分段，用于驱动 AI 视频生成：

```
0-3秒：<location>咖啡厅角落</location>，近景，<role>小明</role>低头看手机，眉头紧锁，手指不停滑动屏幕。
<n>3-6秒：<location>咖啡厅门口</location>，全景，门铃响，<role>小红</role>推门走入，环顾四周。
<n>6-9秒：<location>咖啡厅</location>，中景，<role>小红</role>看到小明，微笑着走过去，在他对面坐下。
<n>9-12秒：<location>咖啡厅角落</location>，近景，<role>小明</role>抬头，表情从焦虑变为惊喜。
```

### 标签说明

| 标签 | 用途 | 示例 |
|-----|------|------|
| `<location>地点</location>` | 场景标记 | `<location>昏暗的卧室</location>` |
| `<role>角色名</role>` | 角色标记 | `<role>小明</role>` |
| `<voice>角色名</voice>` | 画外音/旁白 | `<voice>旁白</voice>` |
| `<n>` | 时间段分隔符 | 段与段之间 |

### video_prompt 写作要点

- `<location>` 要具体，不要只写"室内"，要写"昏暗的卧室"或"阳光明媚的咖啡厅"
- `<role>` 写角色名即可，外观由角色库自动关联
- `<action>` 要具体可执行，不要写抽象概念（"感到悲伤"→"眼眶泛红，咬紧嘴唇"）
- 每段 3 秒，动作要匹配时间长度（3秒内不可能完成"跑过整条街"）

## 场景关联规则

- 优先使用 `read_storyboard_context` 返回的 `scenes` 列表
- `location + time` 可明确匹配时，**必须**回填正确 `scene_id`
- **不要凭空生成不存在的场景 ID**
- 如果剧本内容明显落在已有场景中，不要重复创造新场景描述

## 角色绑定规则

- `character_ids` 必须从 `read_storyboard_context` 返回的角色列表中选择
- 一个镜头可以没有角色（空镜头），也可以绑定多个角色
- 只要镜头里有明确出场、被看见、发生动作或说话的角色，都应绑定
- 纯环境镜头、空镜头、物件镜头传空数组 `[]`

## 时长分配参考

| 镜头类型 | 建议时长 | 说明 |
|---------|---------|------|
| 大远景/远景 | 2-3秒 | 建立场景，不宜过长 |
| 中景对白 | 3-5秒 | 展示交流 |
| 特写表情 | 2-4秒 | 强调情绪 |
| 动作镜头 | 2-4秒 | 保持节奏 |
| 情感高潮 | 4-8秒 | 给观众感受的时间 |
| 转场镜头 | 1-2秒 | 快速过渡 |

## 旁白处理规则

- 若存在旁白，统一写入 `dialogue` 字段
- 格式：`旁白：内容`
- 旁白标记在 video_prompt 中使用 `<voice>旁白</voice>` 标签

## 质量要求

- `description` 要适合**人读**，`video_prompt` 要适合**模型生成**，二者不要互相替代
- `image_prompt` 要突出单帧构图、角色外观、环境和光线
- `video_prompt` 要突出时间推进、动作变化、镜头语言
- `bgm_prompt` 和 `sound_effect` 用简洁短语即可，但不能空泛到只有"紧张""悲伤"
  - ✅ "紧张的心跳声渐强，低频弦乐铺垫"
  - ❌ "紧张"

## 常见问题处理

| 问题 | 处理方法 |
|-----|---------|
| 剧本过长 | 按场景分批处理，确保每个场景拆解完整 |
| 动作复杂 | 拆分为多个短镜头，每个镜头一个核心动作 |
| 对白密集 | 在对白间插入反应镜头（听者的表情变化） |
| 场景缺少视觉描述 | 根据角色和情节推断合理的视觉呈现 |
| 闪回/梦境 | 在 title 中标注（如"噩梦闪回"），atmosphere 中强调虚幻感 |
| 多人对话 | 用过肩镜头/正反打切换，每个角色特写至少一个镜头 |

## 质量自检清单

- [ ] 每个镜头是否只聚焦单一动作
- [ ] video_prompt 是否按 3 秒分段且使用正确标签
- [ ] scene_id 是否从已有场景中匹配，未凭空编造
- [ ] character_ids 是否绑定了所有出场角色
- [ ] bgm_prompt / sound_effect 是否具体（不是"紧张""悲伤"这种空泛词）
- [ ] 旁白是否统一在 dialogue 中使用"旁白：内容"格式
- [ ] 镜头之间是否有叙事连续性，不跳跃不遗漏
