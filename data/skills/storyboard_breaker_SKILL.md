# 分镜拆解专家 — 专业技能指南

## 核心工作流程

1. **读取上下文**：调用 `read_storyboard_context` 获取剧本、角色、场景信息
2. **逐场景拆解**：将每个剧本场景拆解为多个镜头
3. **生成提示词**：为每个镜头生成 imagePrompt 和 videoPrompt
4. **保存结果**：调用 `save_storyboards` 批量保存

## 分镜设计原则

### 镜头选择
- **开场**：用远景或大远景建立场景
- **对白**：中景或过肩镜头，重要对白切特写
- **动作**：中景跟随动作，关键瞬间切特写
- **情绪**：特写捕捉表情变化
- **转场**：远景/大远景作为场景间的呼吸

### 摄影角度的情感含义
- **仰角**：权力、威严、压迫感
- **俯角**：弱小、无助、客观审视
- **荷兰角**：不安、失衡、疯狂
- **平视**：自然、平等、日常

### 运镜方式
- **推/拉**：引导注意力，推=强调，拉=揭示
- **摇/移**：跟随动作，展示空间关系
- **手持**：紧张、纪实感、临场感
- **稳定器**：流畅、优雅、梦境感

## Video Prompt XML规范

每个镜头的 videoPrompt 必须使用 XML 标签格式：

```xml
<location>具体地点描述</location>
<role>角色外观描述（性别、年龄、服装、特征）</role>
<action>具体的动作描述</action>
<atmosphere>氛围和光线描述</atmosphere>
```

### 注意事项
- `<location>` 要具体，不要只写"室内"
- `<role>` 要包含视觉外观，不要只写名字
- `<action>` 要具体可执行，不要写抽象概念
- `<atmosphere>` 要包含光线和色调信息

## 时长分配参考

| 镜头类型 | 建议时长 |
|---------|---------|
| 大远景/远景 | 2-3秒 |
| 中景对白 | 3-5秒 |
| 特写表情 | 2-4秒 |
| 动作镜头 | 2-4秒 |
| 情感高潮 | 4-8秒 |
| 转场镜头 | 1-2秒 |

## 常见问题处理

- **剧本过长**：按场景分批处理，确保每个场景拆解完整
- **动作复杂**：拆分为多个短镜头，每个镜头一个核心动作
- **对白密集**：在对白间插入反应镜头（听者的表情）
- **场景缺少视觉描述**：根据角色和情节推断合理的视觉呈现
