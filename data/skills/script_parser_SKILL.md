# 剧本解析专家 — 专业技能指南

## 核心工作流程

1. **读取上传文本**：调用 `read_uploaded_text` 获取上传的剧本文本
2. **分析剧本结构**：识别集数划分、推断题材和风格
3. **场次拆分**：将每集拆分为多个场次（场景变化=新场次）
4. **角色提取**：识别所有有名角色，判断主配角
5. **场景提取**：识别所有唯一场景地点
6. **道具提取**：识别关键剧情道具
7. **保存解析结果**：调用 `save_parsed_script` 一次性保存所有结构化数据

## 集数拆分策略

- 识别标记：`第N集`、`第N章`、`Episode N`、`EP.N`、`===`分隔线等
- 若无明确集数标记，整篇文本视为1集
- 每集包含标题和完整内容

## 场次拆分规范（场次）

场次是比集更细的粒度，由地点或时间变化来界定。

### 场次识别方法
- 显式标记：`内景/外景`、`日/夜`、场景标题行
- 隐式推断：地点变化、时间跳跃、氛围转换
- 无法确定边界时，宁可不拆也不要错误拆分

### 每个场次包含
| 字段 | 说明 | 要求 |
|-----|------|------|
| sceneNumber | 场次编号 | 从1开始递增 |
| location | 场景地点 | 必填，越具体越好 |
| timeOfDay | 时间段 | day/night/dawn/dusk/morning/afternoon/evening |
| description | 场景简述 | 1句话描述环境和氛围 |
| content | 该场次文本 | 原文对应部分，不摘要 |

### 重要规则
- **episode.content 必须保留完整的原始文本**（不截断、不摘要）
- **scenes[].content 只包含该场次对应的文本片段**
- 若无法清晰拆分场次，scenes 可为空数组

## 角色提取规范

### 提取字段

| 字段 | 说明 | 要求 |
|-----|------|------|
| name | 角色名 | 有名用名，无名用身份代称 |
| role | 角色定位 | protagonist/supporting/minor |
| gender | 性别 | male/female/unknown |
| description | 描述 | 1-2句外貌和性格简述 |

### 角色分级

| 定位 | 判断标准 |
|-----|---------|
| protagonist (主角) | 出场最多、推动核心剧情 |
| supporting (配角) | 有固定名字和多次出场 |
| minor (龙套) | 只出现一两次，对剧情影响小 |

## 场景提取规范

### 提取字段

| 字段 | 说明 | 要求 |
|-----|------|------|
| location | 地点名称 | 必填，越具体越好 |
| timeOfDay | 时间段 | day/night/dawn/dusk/morning/afternoon/evening |
| description | 场景描述 | 环境氛围、关键视觉元素 |

### 去重规则
- 同一地点+同一时间段 = 同一场景
- 同一地点不同时间 = 不同场景（如"咖啡馆·日"和"咖啡馆·夜"）
- 场景列表是全局去重的（跨集去重）

## 道具提取规范

| 字段 | 说明 |
|-----|------|
| name | 道具名称 |
| description | 外观和用途描述 |

- **只提取对剧情有推动作用的道具**（关键信物、武器、线索等）
- 背景性道具（桌上的杯子等）不单独提取

## 输出格式

调用 `save_parsed_script` 工具，包含以下参数：
```json
{
  "title": "项目名称",
  "genre": "甜宠",
  "style": "realistic",
  "totalEpisodes": 3,
  "episodes": [
    {
      "title": "第1集：初遇",
      "content": "完整原文...",
      "scenes": [
        {
          "sceneNumber": 1,
          "location": "咖啡馆",
          "timeOfDay": "afternoon",
          "description": "温馨的咖啡馆内部",
          "content": "该场次对应的原文..."
        }
      ]
    }
  ],
  "characters": [
    { "name": "林小雨", "role": "protagonist", "gender": "female", "description": "25岁白领女孩" }
  ],
  "scenes": [
    { "location": "咖啡馆", "timeOfDay": "afternoon", "description": "温馨的咖啡馆" }
  ],
  "props": [
    { "name": "平安符", "description": "母亲遗物" }
  ],
  "summary": "1-2句话的故事概要"
}
```

## 题材推断关键词

| 题材 | 关键词 |
|-----|-------|
| 都市 | 公司、职场、城市、公寓、地铁 |
| 古装 | 朝、皇宫、太后、将军、武功 |
| 悬疑 | 案件、凶手、侦探、线索、真相 |
| 科幻 | 未来、机器人、AI、太空、星际 |
| 甜宠 | 总裁、霸总、甜、宠、恋爱 |
| 复仇 | 复仇、报仇、陷害、背叛 |
| 励志 | 梦想、奋斗、逆袭、坚持 |
| 校园 | 学校、同学、老师、考试、校园 |

## 常见问题处理

| 问题 | 处理方法 |
|-----|---------|
| 无法识别集数边界 | 视为1集 |
| 场次边界模糊 | 宁可不拆，不要错误拆分 |
| 角色性别不明确 | 设为"unknown" |
| 道具是否关键不确定 | 不提取，宁可漏掉不要多提 |
| 新旧格式兼容 | scenes/characters/props 可为空数组 |

## 质量自检清单

- [ ] episode.content 是否保留了完整原始文本
- [ ] 场次拆分是否合理（地点/时间变化才拆）
- [ ] 主角是否正确识别为 protagonist
- [ ] 场景是否去重（同地点同时段不重复）
- [ ] 道具是否只包含剧情关键道具
- [ ] 题材和风格是否正确推断
